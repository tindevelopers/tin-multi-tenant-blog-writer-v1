## Naming Convention

All frontend integration guides follow this naming pattern:
```
FRONTEND_INTEGRATION_TESTING_GUIDE_X.Y.Z.md
```

Where `X.Y.Z` is the semantic version number (e.g., `1.3.6`).

**When creating a new version:**
1. Copy the latest version file to `FRONTEND_INTEGRATION_TESTING_GUIDE_X.Y.Z.md`
2. Update version number in header
3. Update changelog
4. Update this index file
5. Update `FRONTEND_INTEGRATION_TESTING_GUIDE.md` (redirect file)
